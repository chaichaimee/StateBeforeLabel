StateBeforeLabel


Obtenez l'état en premier. Passez ceux qui ne comptent pas.


auteur : chai chaimee

URL : https://github.com/chaichaimee/StateBeforeLabel


Description

Avez-vous déjà parcouru un long formulaire avec la touche Tab, attendu d'entendre l'étiquette complète d'un contrôle, pour découvrir à la toute fin qu'il était désactivé ou déjà coché ? Ce court délai s'additionne, obligeant votre cerveau à réévaluer chaque contrôle après avoir entendu son étiquette plutôt qu'avant.

State Before Label inverse l'ordre d'annonce. Il dit d'abord les informations d'état les plus importantes, puis le nom, afin que vous puissiez décider instantanément de vous arrêter et d'interagir ou de continuer. Fini d'écouter une longue étiquette de bouton pour n'entendre "Indisponible" qu'à la fin.

Chaque catégorie d'état possède son propre interrupteur marche/arrêt dans les paramètres de NVDA, vous permettant de contrôler exactement quels états bénéficient d'un traitement prioritaire.


Fonctionnalités

Cases à cocher, boutons radio et éléments de menu avec case

Problème : Lorsque l'étiquette d'une case à cocher est longue ou descriptive, NVDA annonce traditionnellement d'abord le nom, puis l'état de coche. Au moment où vous entendez "coché", vous avez déjà investi de l'attention pour comprendre ce qu'est le contrôle.

Avant : "J'accepte les conditions générales et la politique de confidentialité de ce site Web, y compris toutes les sous-sections relatives à la collecte de données. Coché"

Après : "Coché. J'accepte les conditions générales et la politique de confidentialité de ce site Web, y compris toutes les sous-sections relatives à la collecte de données."

Lorsqu'il n'est pas coché, l'extension annonce l'état négatif : "Non coché" avant l'étiquette. L'annonce de rôle par défaut de NVDA comme "case à cocher" est naturellement supprimée pour que l'état ressorte clairement sans redondance.

Exemple concret — Paramètres vocaux de NVDA : Il s'agit d'un panneau que vous visitez probablement souvent. Voici à quoi ressemblent les cases à cocher avec State Before Label actif :

Avant :

  • Augmentation de la vitesse. Case à cocher. Non coché. Alt+t
  • Utiliser le système de sortie audio moderne (WASAPI). Case à cocher. Coché
  • Changement automatique de langue (lorsque pris en charge). Case à cocher. Non coché
  • Changement automatique de dialecte (lorsque pris en charge). Case à cocher. Non coché
  • Signaler les changements de langue lors de la lecture. Case à cocher. Non coché. Alt+g
  • Se fier à la langue de la voix lors du traitement des caractères et symboles. Case à cocher. Coché
  • Données du Consortium Unicode (y compris les emojis). Case à cocher. Coché
  • Descriptions différées pour les caractères lors du déplacement du curseur. Case à cocher. Non coché. Alt+d
  • Dire "Maj" avant les majuscules. Case à cocher. Non coché. Alt+c
  • Bip pour les majuscules. Case à cocher. Non coché. Alt+b
  • Utiliser la fonctionnalité d'orthographe si prise en charge. Case à cocher. Non coché. Alt+s

Après — avec State Before Label :

  • Non coché. Augmentation de la vitesse. Alt+t
  • Co ché. Utiliser le système de sortie audio moderne (WASAPI)
  • Non coché. Changement automatique de langue (lorsque pris en charge)
  • Non coché. Changement automatique de dialecte (lorsque pris en charge)
  • Non coché. Signaler les changements de langue lors de la lecture. Alt+g
  • Co ché. Se fier à la langue de la voix lors du traitement des caractères et symboles
  • Co ché. Données du Consortium Unicode (y compris les emojis)
  • Non coché. Descriptions différées pour les caractères lors du déplacement du curseur. Alt+d
  • Non coché. Dire "Maj" avant les majuscules. Alt+c
  • Non coché. Bip pour les majuscules. Alt+b
  • Non coché. Utiliser la fonctionnalité d'orthographe si prise en charge. Alt+s

Remarquez comment le mot "Case à cocher" a disparu de chaque ligne. Vous n'entendez que l'état suivi de l'étiquette et du raccourci. Cela transforme une page de paramètres encombrée en une liste claire et immédiatement analysable où votre cerveau traite d'abord l'état activé/désactivé, puis le nom de la fonction. Parcourir 11 cases à cocher avec Tab devient nettement plus rapide lorsque chacune suit le même modèle précis.


Boutons bascules

Problème : Les boutons bascules comme Gras, Italique ou Muet indiquent visuellement leur état par surbrillance, mais les utilisateurs de lecteurs d'écran doivent entendre d'abord l'étiquette avant de savoir si la fonction est activée ou désactivée.

Avant : "Gras. Bouton bascule. Enfoncé"

Après : "Enfoncé. Gras"

Lorsque la bascule est désactivée, "Non enfoncé" est dit avant le nom du bouton. L'annonce de rôle est également simplifiée afin que l'attention reste sur l'état puis l'identité.

Exemple concret — Paramètres Windows > Comptes > Options de connexion : Cette page contient plusieurs boutons bascules qui contrôlent le comportement de Windows au redémarrage ou à la reconnexion. Voici comment l'un d'eux se lit :

Avant : "Enregistrer automatiquement mes applications redémarrables et les redémarrer lorsque je me reconnecte. Bouton bascule. Non enfoncé"

Après : "Non enfoncé. Enregistrer automatiquement mes applications redémarrables et les redémarrer lorsque je me reconnecte"

L'étiquette est longue, mais avec State Before Label vous entendez d'abord l'état. Si la bascule est déjà activée, vous entendez "Enfoncé" et continuez. Si elle est désactivée et que vous voulez l'activer, vous le savez immédiatement et appuyez sur la barre d'espace. Pas d'attente à travers une longue phrase pour arriver au point de décision.


Éléments d'arborescence (Développé/Réduit)

Problème : Naviguer dans une structure arborescente dans l'Explorateur de fichiers ou les panneaux de configuration nécessite de savoir si une branche est ouverte ou fermée. Entendre d'abord le nom du dossier vous oblige à vous rappeler où vous êtes, puis à traiter l'état d'expansion après.

Avant : "Documents. Développé. Niveau 2"

Après : "Développé. Documents"

Les branches réduites annoncent "Réduit" en premier, vous donnant un indice immédiat pour appuyer sur la flèche droite si vous voulez explorer plus profondément, sans attendre d'entendre le nom complet.


Boutons avec état Développé/Réduit

Problème : Les sites Web et applications modernes utilisent souvent des boutons standard comme déclencheurs de menus déroulants ou en-têtes d'accordéon avec aria-expanded. Ces boutons ressemblent à des boutons normaux pour NVDA, mais leur état d'expansion est essentiel pour comprendre si un menu ou une section de contenu est actuellement ouvert.

Avant : "Paramètres. Bouton" (aucune indication que le menu déroulant est ouvert)

Après : "Développé. Paramètres"

Cette fonction est optionnelle via les paramètres. Lorsqu'elle est activée, tout rôle de Bouton portant l'état Développé ou Réduit annoncera d'abord l'état, vous donnant la même conscience de type arborescence pour les menus, les accordéons de FAQ et les sections extensibles.


État sélectionné pour les onglets et les éléments de liste

Problème : Lorsque vous parcourez des onglets dans une boîte de dialogue ou naviguez dans une longue liste de fichiers, savoir quel élément est actuellement sélectionné nécessite d'entendre d'abord le nom, puis "Sélectionné". Dans une liste de dizaines d'éléments, cela ajoute une charge cognitive.

Avant : "Général. Onglet. Sélectionné"

Après : "Sélectionné. Général"

Ce paramètre est désactivé par défaut pour garder l'expérience minimale. Activez-le lorsque vous travaillez fréquemment avec des interfaces à onglets ou de longues listes sélectionnables où la conscience de la sélection actuelle est importante.


État indisponible (Global)

Problème : Les contrôles désactivés apparaissent partout : boutons grisés, champs d'édition verrouillés, éléments de menu indisponibles. Entendre un long nom pour découvrir que le contrôle est inutilisable gaspille des frappes et de l'énergie mentale.

Avant : "Soumettre la demande. Bouton. Indisponible"

Après : "Indisponible. Soumettre la demande"

Cette vérification s'exécute globalement pour chaque type de contrôle. Tout objet portant l'état Indisponible l'annoncera en premier, vous permettant de passer immédiatement à côté. Le rôle du contrôle et les autres états sont gérés proprement afin que rien ne soit annoncé deux fois.


État lecture seule pour les champs d'édition

Problème : Vous arrivez sur un champ d'édition prêt à taper, pour n'entendre "Lecture seule" qu'à la fin de l'annonce. Cela interrompt le flux de travail et crée une confusion quant à savoir si le champ est réellement modifiable.

Avant : "Adresse e-mail. Édition. Lecture seule"

Après : "Lecture seule. Adresse e-mail"

L'extension ajoute "Lecture seule" avant le nom du champ tout en préservant tout le comportement standard de NVDA : l'annonce de la sélection de texte, la position du curseur et la lecture des valeurs fonctionnent exactement comme prévu.


Utilisation

  1. Ouvrez le menu NVDA > Préférences > Paramètres > State Before Label.
  2. Cochez ou décochez chaque catégorie d'état selon vos préférences.
  3. Appuyez sur OK. Les modifications prennent effet immédiatement sans redémarrer NVDA.

Toutes les fonctionnalités fonctionnent automatiquement lorsque vous déplacez le focus avec Tab, les flèches ou la navigation souris. Aucun raccourci clavier supplémentaire à mémoriser.
