StateBeforeLabel


Obtén el estado primero. Omite los que no importan.


autor: chai chaimee

URL: https://github.com/chaichaimee/StateBeforeLabel


Descripción

¿Alguna vez has navegado con tabulador a través de un formulario largo, esperando escuchar la etiqueta completa de un control, solo para descubrir al final que estaba deshabilitado o ya estaba marcado? Ese breve retraso se acumula, obligando a tu cerebro a reevaluar cada control después de escuchar su etiqueta en lugar de antes.

State Before Label invierte el orden de anuncio. Primero habla la información de estado más importante, luego el nombre, para que puedas decidir al instante si detenerte e interactuar o seguir adelante. No más escuchar una etiqueta larga de botón solo para oír "No disponible" al final.

Cada categoría de estado tiene su propio interruptor de activación/desactivación en la configuración de NVDA, para que controles exactamente qué estados reciben tratamiento prioritario.


Características

Casillas de verificación, botones de opción y elementos de menú con casilla

Problema: Cuando la etiqueta de una casilla de verificación es larga o descriptiva, NVDA tradicionalmente anuncia primero el nombre y luego el estado de marcado. Para cuando escuchas "marcado", ya has invertido atención en comprender qué es el control.

Antes: "Acepto los términos y condiciones y la política de privacidad de este sitio web, incluidas todas las subsecciones relacionadas con la recopilación de datos. Marcado"

Después: "Marcado. Acepto los términos y condiciones y la política de privacidad de este sitio web, incluidas todas las subsecciones relacionadas con la recopilación de datos."

Cuando no está marcado, el complemento habla el estado negativo: "No marcado" antes de la etiqueta. El anuncio de rol predeterminado de NVDA, como "casilla de verificación", se suprime naturalmente para que el estado destaque claramente sin redundancia.

Ejemplo del mundo real — Configuración de voz de NVDA: Este es un panel que probablemente visitas a menudo. Así es como aparecen las casillas de verificación con State Before Label activo:

Antes:

  • Aumento de velocidad. Casilla de verificación. No marcado. Alt+t
  • Usar sistema moderno de salida de audio (WASAPI). Casilla de verificación. Marcado
  • Cambio automático de idioma (cuando sea compatible). Casilla de verificación. No marcado
  • Cambio automático de dialecto (cuando sea compatible). Casilla de verificación. No marcado
  • Informar cambios de idioma al leer. Casilla de verificación. No marcado. Alt+g
  • Confiar en el idioma de la voz al procesar caracteres y símbolos. Casilla de verificación. Marcado
  • Datos del Consorcio Unicode (incluidos emoji). Casilla de verificación. Marcado
  • Descripciones retrasadas para caracteres al mover el cursor. Casilla de verificación. No marcado. Alt+d
  • Decir "Mayús" antes de mayúsculas. Casilla de verificación. No marcado. Alt+c
  • Pitar para mayúsculas. Casilla de verificación. No marcado. Alt+b
  • Usar funcionalidad de ortografía si es compatible. Casilla de verificación. No marcado. Alt+s

Después — con State Before Label:

  • No marcado. Aumento de velocidad. Alt+t
  • Marcado. Usar sistema moderno de salida de audio (WASAPI)
  • No marcado. Cambio automático de idioma (cuando sea compatible)
  • No marcado. Cambio automático de dialecto (cuando sea compatible)
  • No marcado. Informar cambios de idioma al leer. Alt+g
  • Marcado. Confiar en el idioma de la voz al procesar caracteres y símbolos
  • Marcado. Datos del Consorcio Unicode (incluidos emoji)
  • No marcado. Descripciones retrasadas para caracteres al mover el cursor. Alt+d
  • No marcado. Decir "Mayús" antes de mayúsculas. Alt+c
  • No marcado. Pitar para mayúsculas. Alt+b
  • No marcado. Usar funcionalidad de ortografía si es compatible. Alt+s

Observa cómo la palabra "Casilla de verificación" ha desaparecido de cada línea. Solo escuchas el estado seguido de la etiqueta y el atajo. Esto transforma una página de configuración abarrotada en una lista limpia y escaneable al instante, donde tu cerebro procesa primero el estado de activación/desactivación y luego el nombre de la función. Navegar con tabulador a través de 11 casillas de verificación se vuelve significativamente más rápido cuando cada una sigue el mismo patrón nítido.


Botones de alternancia

Problema: Los botones de alternancia como Negrita, Cursiva o Silenciar indican visualmente su estado mediante resaltado, pero los usuarios de lectores de pantalla deben escuchar primero la etiqueta antes de saber si la función está activada o desactivada.

Antes: "Negrita. Botón de alternancia. Presionado"

Después: "Presionado. Negrita"

Cuando la alternancia está desactivada, se dice "No presionado" antes del nombre del botón. El anuncio de rol se simplifica de manera similar para que el enfoque permanezca en el estado y luego en la identidad.

Ejemplo del mundo real — Configuración de Windows > Cuentas > Opciones de inicio de sesión: Esta página contiene varios botones de alternancia que controlan cómo se comporta Windows cuando reinicias o inicias sesión nuevamente. Así se lee uno de ellos:

Antes: "Guardar automáticamente mis aplicaciones reiniciables y reiniciarlas cuando vuelva a iniciar sesión. Botón de alternancia. No presionado"

Después: "No presionado. Guardar automáticamente mis aplicaciones reiniciables y reiniciarlas cuando vuelva a iniciar sesión"

La etiqueta es larga, pero con State Before Label escuchas el estado primero. Si la alternancia ya está activada, escuchas "Presionado" y sigues adelante. Si está desactivada y quieres activarla, lo sabes de inmediato y presionas la barra espaciadora. Sin esperar a una oración larga para llegar al punto de decisión.


Elementos de vista de árbol (Expandido/Colapsado)

Problema: Navegar por una estructura de árbol en exploradores de archivos o paneles de configuración requiere saber si una rama está abierta o cerrada. Escuchar primero el nombre de la carpeta te obliga a recordar dónde estás y luego procesar el estado de expansión.

Antes: "Documentos. Expandido. Nivel 2"

Después: "Expandido. Documentos"

Las ramas colapsadas anuncian "Colapsado" primero, dándote una señal inmediata para presionar la flecha derecha si quieres explorar más profundamente, sin esperar a escuchar el nombre completo.


Botones con estado Expandido/Colapsado

Problema: Los sitios web y aplicaciones modernos suelen usar botones estándar como activadores de menús desplegables o encabezados de acordeón con aria-expanded. Estos botones parecen botones normales para NVDA, pero su estado de expansión es fundamental para comprender si un menú o sección de contenido está abierto actualmente.

Antes: "Configuración. Botón" (sin indicación de que el desplegable esté abierto)

Después: "Expandido. Configuración"

Esta función es optativa mediante la configuración. Cuando está activada, cualquier rol de Botón que tenga estado Expandido o Colapsado anunciará primero el estado, brindándote la misma conciencia similar a la vista de árbol para menús, acordeones de preguntas frecuentes y secciones expandibles.


Estado seleccionado para pestañas y elementos de lista

Problema: Al recorrer pestañas en un diálogo o navegar por una larga lista de archivos, saber qué elemento está seleccionado actualmente requiere escuchar primero el nombre y luego "Seleccionado". En una lista de docenas de elementos, esto añade carga cognitiva.

Antes: "General. Pestaña. Seleccionado"

Después: "Seleccionado. General"

Esta configuración está desactivada por defecto para mantener la experiencia mínima. Actívala cuando trabajes frecuentemente con interfaces de pestañas o listas seleccionables largas donde la conciencia de la selección actual sea importante.


Estado no disponible (Global)

Problema: Los controles deshabilitados aparecen en todas partes: botones atenuados, campos de edición bloqueados, elementos de menú no disponibles. Escuchar un nombre largo solo para descubrir que el control no es utilizable desperdicia pulsaciones de teclas y energía mental.

Antes: "Enviar solicitud. Botón. No disponible"

Después: "No disponible. Enviar solicitud"

Esta verificación se ejecuta globalmente para cada tipo de control. Cualquier objeto que tenga el estado No disponible lo anunciará primero, permitiéndote saltarlo inmediatamente. El rol del control y otros estados se manejan de manera limpia para que nada se anuncie dos veces.


Estado de solo lectura para campos de edición

Problema: Llegas a un campo de edición listo para escribir, solo para escuchar "Solo lectura" al final del anuncio. Esto interrumpe el flujo de trabajo y crea confusión sobre si el campo es realmente editable.

Antes: "Correo electrónico. Edición. Solo lectura"

Después: "Solo lectura. Correo electrónico"

El complemento antepone "Solo lectura" antes del nombre del campo, preservando todo el comportamiento estándar de NVDA: el anuncio de selección de texto, la posición del cursor y la lectura de valores funcionan exactamente como se espera.


Uso

  1. Abre el Menú de NVDA > Preferencias > Configuración > State Before Label.
  2. Marca o desmarca cada categoría de estado según tus preferencias.
  3. Presiona Aceptar. Los cambios surten efecto inmediatamente sin reiniciar NVDA.

Todas las funciones funcionan automáticamente cuando mueves el enfoque con Tab, teclas de flecha o navegación con el ratón. No hay que memorizar pulsaciones de teclas adicionales.
